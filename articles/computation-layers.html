O computador em Camadas

O computador é uma maquina complexa, uma boa maneira de ter uma visão gral de seu funcionamento é dividi-la em componentes e estudar a responsabilidade de cada um individualmente, e depois entender como esses componentes se integram. Nesse artigo busco dar ao leitor uma visão geral do que acontece quando o programa esta em execução, destacando seus principais componente apatir de um ponto de vista top-down (de cima para baixo), conforme figura a seguir:

O computador possui componentes fisicos e abstratos, hardware e software respectivamente. As sessões abaixo irão detalhar mais a parte abstrata, o software. O hardware será visto apenas como o componete fisico capaz de seguir as instruções do software.

Muitos programadores não sabem oque acontece nas camadas abaixo do programa, como acontece a integração entre programa, sistema operacional e hardware. Esse artigo traz um vislumbre sobre oque ocorre por debaixo dos panos quando o programa esta em execução.

Programas de Usuário

São programas usados pelo usuário final, seja ele um economista organizando seu trabalho em um editor de planilhas, um engenheiro civil construindo um projeto de um novo edifício usando um DAC (desenho assistido por computador), ou um programador escrevendo seus programas em um editor de texto. Esse é motivo pelo qual os computadores existem, para prover suporte a diversas áreas através de programas de usuário. Esses programas são criados usando uma linguagem de programação, que sucintamente falando são maneiras de instruir o hardware a fazer alguma coisa.

Sistema Operacional

Quando criamos nosso programa ele não funciona de forma independente, ele sempre vai interagir com o sistema operacional para pedir suporte, mas oque seria exatamente esse suporte?. São funções já prontas disponíveis em memória que podem ser usadas pelos programas de usuário. Funções que implementam funcionalidades de propósito geral como escrever dados em dispositivo de saída, ler dados de um dispositivo de entrada, alocar memória... Também é responsabilidade do sistema operacional gerenciar a memória, cada programa deve ter uma área de memoria reservada , um programa não pode ler ou escrever dados em uma posição de memoria que esta em uso por outros programas.

Chamadas de Sistema

Quando um programa solicita uma função do sistema operacional ele faz isso através de uma chamada de sistema (syscall), quando isso acontece o programa para de executar e o sistema operacional entra em cena. O programa pode interagir diretamente com o sistema operacional ou através de uma biblioteca padrão disponibilizada junto com o sistema operacional, por exemplo glibc (GNU library C) do linux.

O Kernel

O componente principal do sistema operacional é o kernel (do inglês núcleo), é nele que ficam implementadas todas as interfaces das chamadas de sistemas, e também é ele que gerencia os drivers e a memoria. Na figura acima foi usada um figura que mostra a arquitetura de um sistema operacional linux porem o conceito de kernel esta presente em todos de sistemas operacionais.

Hardware

Todas as camadas acima da camada do hardware só podem ser visualizadas através de abstrações, mas o hardware é algo físico ele pode ser visto, tocado e sentido. É ele o responsável por executar as instruções do software. O Hardware do computador é composto de vários componentes, os principais são: I/O (input/output) ,processador e a memoria. O hardware realiza algumas operações básicas para qualquer computador, como: entrada e saída de dados, processamento e armazenamento. A interação entre esses componentes é basicamente da seguinte forma: o processador obtém dados e instruções da memoria, os dispositivos de entrada guardam dados na memoria e os dispositivos de saída leem dados da memoria e apresentam o resultado da computação para o usuário através de um monitor ou para outro computador através de uma placa de rede. Os cursos de ciência da computação e analise de sistemas não se aprofundam muito no estudo do hardware, geralmente o mais profundo que chegamos é até portas lógicas.

"Dispositivos de hardware são construídos a partir de pacote integrados de portas lógicas. E essas portas, por sua vez, podem ser construídas de portas lógicas elementares como a Nand ou Nor, é claro que cada uma dessas portas consiste de alguns dispositivos de switching. tipicamente implementados por transistors. e cada transistor é construído de - Bem, nós não iremos mais longe, é onde a ciência da computação acaba e a física começa."

A frase acima esta no livro Building a Modern Computer from First Principles, isso significa que detalhes do hardware não estão no escopo de estudo da ciência da computação, sendo assim não precisamos saber como certas tarefas são implementadas, podemos apenas saber oque elas fazem e consumir seus serviços.
